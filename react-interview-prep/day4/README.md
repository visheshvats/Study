# Day 4: useEffect Deep Dive I ✅

Welcome to Day 4! Today you'll master useEffect - the foundation of data fetching in React.

## 📚 What You'll Learn

1. **useEffect Basics** - Run side effects after render
2. **Dependency Arrays** - Control when effects run  
3. **Cleanup Functions** - Prevent memory leaks

## 🗂️ Materials

- **[lesson.md](lesson.md)** - useEffect, dependencies, and cleanup explained
- **exercises/** - 3 hands-on exercises (45 mins total)
  - Exercise 1: useEffect Basics (document title, mount logger)
  - Exercise 2: Cleanup Functions (timers with proper cleanup)
  - Exercise 3: Data Fetching (user fetcher with loading/error)
- **[interview-question.md](interview-question.md)** - Fetch posts with race condition handling
- **[mini-project.md](mini-project.md)** - GitHub User Search (30-60 mins)
- **[recap.md](recap.md)** - Summary and Day 5 preview

## ⏱️ Time Estimate

- Lesson: 15 mins
- Exercises: 45 mins
- Interview Q: 15 mins
- Mini-project: 60 mins
- **Total: ~2-2.5 hours**

## 🚀 Getting Started

1. Read `lesson.md` first
2. Complete exercises 1, 2, and 3 in order
3. Answer the interview question (fetch posts list)
4. Build the GitHub User Search mini-project
5. Review `recap.md`

## ✅ Success Criteria

By the end of Day 4, you should be able to:
- [ ] Use useEffect with different dependency arrays
- [ ] Write cleanup functions for timers and subscriptions
- [ ] Fetch data from APIs with loading/error states
- [ ] Prevent memory leaks and race conditions
- [ ] Implement debounced search
- [ ] Explain when and why effects run
