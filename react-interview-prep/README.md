# 🚀 React Interview Prep - Beginner to Interview-Ready

**Your Level:** Beginner in React  
**Goal:** Become interview-ready for a React developer role  
**Duration:** 7 days of focused, hands-on learning

## 📚 Learning Philosophy

- **Teach like a mentor:** Simple language first, gradually increase depth
- **Learn by doing:** Short explanations → Code → Exercises → Feedback
- **Strict feedback:** When you're wrong, expect corrections and improvements
- **Interview-focused:** Build the thinking and skills interviewers look for

## 🎯 Daily Workflow (Every Session)

1. **3-minute concept lesson** – Focus on what you need today
2. **2-3 micro-exercises** – 5-15 mins each, hands-on practice
3. **1 interview-style question** – Beginner level, with evaluation
4. **1 feature task** – 30-60 mins, implement something real
5. **Code review** – Bugs, anti-patterns, improvements, best practices
6. **Recap + Tomorrow's prep** – What to revise and what's next

## 📅 7-Day Syllabus

### **Day 1: React Fundamentals I**
- JSX syntax and why it exists
- Components (function components)
- Props: passing data down
- **Mini-project:** Personal Card Component

### **Day 2: React Fundamentals II**
- State with `useState`
- Rendering lists with `.map()`
- Keys and why they matter
- **Mini-project:** Dynamic Todo List (add only)

### **Day 3: Events & Forms**
- Event handling in React
- Controlled components (inputs)
- Conditional rendering patterns
- **Mini-project:** Interactive Counter + Form

### **Day 4: Hooks Deep Dive I**
- `useEffect` basics
- Dependency arrays explained
- Cleanup functions (when and why)
- **Mini-project:** Data Fetcher with Loading States

### **Day 5: Hooks Deep Dive II**
- `useRef` for DOM and stable values
- `useMemo` and `useCallback` (when NOT to use)
- Common hook mistakes
- **Mini-project:** Search with Debouncing

### **Day 6: Component Design**
- Lifting state up
- Prop drilling vs Context API
- Composition patterns
- **Mini-project:** Reusable Modal Component

### **Day 7: Performance & Best Practices**
- Re-render causes and prevention
- React Strict Mode gotchas
- Common anti-patterns
- **Mini-project:** Optimize a slow component

## 🛠️ How to Use This Repo

1. Each day has its own folder (`day1/`, `day2/`, etc.)
2. Start with `dayX/lesson.md` for concepts
3. Complete exercises in `dayX/exercises/`
4. Build the mini-project in `projects/dayX-project/`
5. Self-review with the provided checklist
6. Submit your code for review

## ✅ Self-Review Checklist (Use Before Submitting)

Before showing your code, always check:
- [ ] Does it run without errors?
- [ ] Did I handle edge cases mentioned in the task?
- [ ] Are my variable/function names clear?
- [ ] Did I avoid common mistakes mentioned in the lesson?
- [ ] Can I explain WHY I wrote it this way?
